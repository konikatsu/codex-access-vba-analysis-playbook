# Access解析プレイブック

Access `.accdb` / `.mdb` をAIエージェントで解析するときの標準手順です。

作業前に、共通ルールも確認してください。

- [Access作業共通ルール](access-work-common-rules.md)

## 最初の原則

- 元DBを直接変更しない。まずコピーで検証する。
- 起動マクロ、`AutoExec`、`StartUp`、外部同期処理を止める。
- 解析用モジュールは一時的に入れてよいが、業務処理を起動しない状態で使う。
- `LoadFromText`、`Application.Run`、VBEコード操作、コンパイルは別々に切り分ける。
- 外部COMで失敗したら、サンドボックスや権限条件を疑う。
- COM操作後は `CloseCurrentDatabase`、`Quit`、`ReleaseComObject`、GCを行い、ロックファイルが残っていないか見る。

## 標準フロー

1. 対象DB、対象処理、調査目的を確認する。
2. Accessが起動中なら閉じる。
3. `.laccdb` / `.ldb` が残っていないか確認する。
4. 作業用コピーを作る。
5. 起動処理を止める。
6. 解析用モジュールまたは `SaveAsText` でコードと定義を出力する。
7. 出力したテキストを `rg` などで検索する。
8. 修正候補を「現象 / 原因 / 対象 / 修正方針」に分ける。
9. 修正する場合は、トランザクション、再実行時の挙動、親子テーブル整合性を確認する。
10. テスト結果は「何を実行したか」「何が変わったか」「残リスク」を分けて記録する。

## コード書き戻し後の差分確認

`CodeModule.DeleteLines` と `CodeModule.AddFromString` を使ってモジュール全文を書き戻す処理は便利ですが、環境や対象コードによっては想定外の文字列が混入することがあります。

特に、変換処理後に次のような孤立行が入り、VBAの構文エラーになることがあります。

```vb
()
)
```

変換後は、必ず `SaveAsText` で対象モジュールを出力し、適用前との差分を確認します。

検索例:

```powershell
rg -n "^\s*\(\)\s*$|^\s*\)\s*$" "C:\work\access-project\exports\after"
```

孤立した括弧行が複数モジュールに出る場合は、個別に直し続けるより、バックアップへ戻して変換ロジックを修正してから再適用する方が安全です。

## コード置換は最小範囲にする

VBEオブジェクトモデルでコードを置換する場合、モジュール全体を削除して追加し直すより、対象行だけを置換する方が安全です。

避けたい例:

```vb
modCode.DeleteLines 1, lineCount
modCode.AddFromString text
```

推奨:

```vb
modCode.ReplaceLine targetLine, newLine
```

方針:

- まず `CodeModule.Find` や行単位の走査で一致行を探す。
- 一致した行だけ `ReplaceLine` で置換する。
- 置換後に `SaveAsText` で出力し、差分を見る。
- 孤立した `()` や `)` が混入していないか検索する。
- 最後に `RunCommand(126)` でコンパイルする。

## 起動処理を止める理由

Access DBは、開いた瞬間に業務処理が動くことがあります。

例:

- `AutoExec`
- `StartUp`
- メニュー表示
- 外部同期
- 締め処理
- 外部DB接続
- 帳票出力
- ダイアログ表示

これらが動くと、解析前にデータが変わったり、COM操作がダイアログ待ちで止まったりします。  
解析目的ではコードと定義を取り出したいだけなので、起動処理は止めます。

## 静的解析で見るポイント

- フォームイベントが存在するか。
- 計算結果が保存直前に上書きされていないか。
- 新規登録時に採番IDを取得しているか。
- 親子テーブル更新がトランザクション化されているか。
- 削除後再作成の途中失敗で不整合にならないか。
- 確定処理と解除処理が対称か。
- 再実行時に二重登録しないか。
- 繰越、補正、未来日付の扱いが明確か。

## 報告フォーマット

```text
現象:
  画面上またはデータ上で起きること。

原因:
  コード上の直接原因。

対象:
  修正したいフォーム、モジュール、関数、テーブル。

修正方針:
  どの値を正とし、どのタイミングで保存するか。

テスト:
  新規、修正、削除、確定、解除、再実行、異常系。
```
