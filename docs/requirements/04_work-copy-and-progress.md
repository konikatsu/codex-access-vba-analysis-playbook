# 作業コピーと進捗メモで安全に進める

## 目的

Access DB作業では、一度失敗したDBを修復しながら進めると、原因が分からなくなりがちです。

作業コピーと進捗メモで、状態を管理します。

## 作業コピー運用

```text
本体DB
-> step001用コピー
-> 成功したらstep002用コピー
-> 成功したらstep003用コピー
```

失敗したコピーは、`failed001` のように退避するか破棄します。

## フォルダ例

```text
work/
  estimate_progress.md
  step001_list/
    copies/
    exports/
    logs/
    screenshots/
    notes.md
  step002_entry/
    copies/
    exports/
    logs/
    screenshots/
    notes.md
```

## 進捗メモ例

```markdown
# 機能追加 進捗メモ

更新日: 2026-06-22

## 全体進捗

- 5 / 8 完了
- 約65%

## 進捗表

| No | 項目 | 状態 | 完了したこと | 残っていること | 次にやること |
|---:|---|---|---|---|---|
| 1 | テストDB環境 | 完了 | Docker DB起動 | なし | 維持 |
| 2 | 作業コピー運用 | 完了 | 成功コピー方式 | なし | 次コピー作成 |
| 3 | 一覧画面 | 作業中 | 基本表示 | 固定編集ボタン | GUI確認 |
```

## 更新ルール

- 作業開始前に更新する。
- 作業完了後に更新する。
- 詰まったら、実装を止めて詰まりを書く。
- 次の30分でやることを1つに絞る。
