# 作業コピーと進捗メモで安全に進める

## 目的

Access DB作業では、一度失敗したDBを修復しながら進めると、原因が分からなくなりがちです。

作業コピーと進捗メモで、状態を管理します。

## 作業コピー運用

```text
本体DB
-> step001用コピー
-> 成功したらstep002用コピー
-> 成功したらstep003用コピー
```

失敗したコピーは、`failed001` のように退避するか破棄します。

作業コピー運用へ移行した後は、ユーザーへ確認対象として元DBを案内しません。
確認してもらうMDBは、最新の確認用作業コピーを1つだけ先頭に明記します。

## フォルダ例

```text
work/
  estimate_progress.md
  step001_list/
    copies/
    exports/
    logs/
    screenshots/
    notes.md
  step002_entry/
    copies/
    exports/
    logs/
    screenshots/
    notes.md
```

## 進捗メモ例

```markdown
# 機能追加 進捗メモ

更新日: 2026-06-22

## 現在のDB状態

- 今触るDB:
- 最後に成功したコピー:
- ユーザー確認対象:
- 元DB:
- バックアップ:
- データDB:

## 全体進捗

- 5 / 8 完了
- 約65%

## 進捗表

| No | 項目 | 状態 | 完了したこと | 残っていること | 次にやること |
|---:|---|---|---|---|---|
| 1 | テストDB環境 | 完了 | Docker DB起動 | なし | 維持 |
| 2 | 作業コピー運用 | 完了 | 成功コピー方式 | なし | 次コピー作成 |
| 3 | 一覧画面 | 作業中 | 基本表示 | 固定編集ボタン | GUI確認 |
```

## ユーザー確認対象の案内

ユーザーから「どのMDBを確認すればよいか」と聞かれた場合は、回答前に進捗メモの次の項目を確認します。

- 今触るDB
- 最後に成功したコピー
- ユーザー確認対象
- 元DB
- バックアップ
- データDB

回答の先頭には、確認で開くMDBを1つだけ明記します。
元DB、成功コピー、バックアップ、データDBは補足として分けます。

案内例:

```text
確認で開くMDB:
  ...\work_xxx\copies\step002_check.mdb

補足:
- 最後に成功したコピー: ...\work_xxx\copies\step001_success.mdb
- 元DB: ...\app.mdb
- バックアップ: ...\backup\app_before_work.mdb
- データDB: ...\data\app_data.mdb
```

注意:

- 作業コピー運用へ移行した後は、元DBを確認対象として案内しない。
- 元DBやバックアップを、確認で開くMDBのように書かない。
- フロントMDBの作業コピーと、リンク先データMDBは役割を分けて書く。
- 進捗メモにユーザー確認対象が書けない場合は、まだ案内しない。

## 更新ルール

- 作業開始前に更新する。
- 作業完了後に更新する。
- 詰まったら、実装を止めて詰まりを書く。
- 次の30分でやることを1つに絞る。
